PageRank

Imagine you are at the beginning of the Internet and trying to find the official website of ebay, which you overheard some people talking about. So you search, "ebay". What you are looking for is obviously www.ebay.com, but what comes up is a bunch of random websites that mention it one way or another. In this hypothetical example, what happened is the search engine tried to match the query terms to the billions of web pages naively. It mistakenly ranked noise or spam over the actual website you were looking for. Now to address this issue, you think about the hyperlinked structure of the web and come up with an idea of leveraging in-link and out-link counts to help you rank pages, in addition to literal query term matches. This helps but you notice that there seem to be a lot of pages that specifically target these simple count-based ranking algorithms and manipulate the results. Now you think about the problem a bit harder and come up with two criteria to judge whether a page is important or not.

  • higher the count of in-links, higher the page's score
  • higher the rank of the page where the in-link is from, higher the page's score
In other words, "a page has high rank if the sum of the ranks of its backlinks is high"1.

The above intuition can be formalized as follows:

PR(u) = λN + (1−λ) ⁢ ∑ v∈Bu PR(v) Lv

where:

λ probability of jumping to a random page (e.g. 0.15)
N total number of pages in the graph
1−λ probability of following an out-link
∑ v∈Bu sum over all pages v that link to page u (set Bu)
PR(v) current PageRank score of page v
Lv number of out-links from page v

Ignore the first part of the equation for now (since we didn't talk about it yet). The above two bullet points directly map to the second part. The count of pages that link to page u is represented by Bu and the rank of each page that links to u is represented by PR(v). In simple terms, the second part is essentially saying, "take all of the pages that link to page u, calculate each's contribution by dividing each's rank by the number of out-links each has and sum them up."

To understand the first part intuitively, let's introduce a random web surfer who hops from page to page by following out-links, with an occasional “random jump” (e.g., a person might visit different pages on a news site and then jump to a youtube). The first part of the above equation accounts for this random jumping. To be more precise, given there is a 1/N chance of coming to the page u randomly, the contribution of the random jump will be λ/N. In other words, the rank of page u is influenced not only by link-following (two bullet points above) but also some random jumps.

Moving higher/from higher level of abstraction, the above per-page equation can be expressed as a matrix equation for a vector of PageRank values:

Rk + 1 = T ⁢ Rk

where:

Rk + 1 new ranks
T transition probabilities for random surfer model
Rk old ranks

You can think of Rk as the vector of page ranks after k iterations. (R0 usually starts uniform, e.g., 1/N). In each iteration, we update the ranks using the transition probabilities matrix T, where Tij represents the probability of going from page i to page j and is equal to: Tij = λN + (1−λ) ⁢ 1 Lv

When Rk + 1=Rk, we have reached the stationary distribution, which is the final ranks of the pages.

To summarize, PageRank is an example of query-independent metadata that can improve ranking for web search. It is relatively objective and addresses the above mentioned issues well.
