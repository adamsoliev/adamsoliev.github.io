Computer Architecture

Hardware Organization

  • Architecture
  • Instruction Set Architecture (ISA)
    • Interface between hardware and software
    • Types
      • CISC
      • RISC
    • Includes instruction formats, addressing modes, registers
  • Memory Hierarchy
    • Multiple levels of memory
      • Registers (fastest, smallest, most expensive)
      • Cache (L1, L2, L3)
      • Main memory (RAM)
      • Disk (HDD, SSD)
      • Distributed file systems
    • Cache coherence and memory management
    • Virtual memory
  • Pipelining
    • Pipeline Stages
      • Fetch - Get instruction from memory
      • Decode - Determine instruction type and operands
      • Execute - Perform ALU operation
      • Memory Access - Load/store data if needed
      • Write Back - Store results in registers
    • Hazards
      • Structural - Hardware resource conflicts
      • Data - Instruction depends on result of prior instruction
      • Control - Delays from branches and jumps
    • Instruction-level parallelism
  • Parallel processing
    • Multiple processors working simultaneously
    • Types
      • SIMD (Single Instruction Multiple Data)
      • MIMD (Multiple Instruction Multiple Data)
      • Multicore processors
    • Thread-level parallelism
  • I/O devices
  • Performance optimization
  • Power and energy management
  • System integration
  • Security
  • Reliability and Fault Tolerance
